# 貯金アプリ v0.9.11.1

スマートフォン向け・静的Webアプリです。`index.html` をWebサーバー経由で開くか、GitHub Pagesで公開してください。

- 記録・設定はブラウザの localStorage に保存されます。
- PWAのオフラインキャッシュは、HTTPS または localhost で有効になります。
- 実際の銀行残高は扱いません。

## v0.9.7 貯金カレンダー

ホームの「貯金カレンダー」から、保存済みの貯金を月別・日別に振り返れます。通常・クイック・リベンジ貯金を同じ「貯金」として日ごとに合計し、日付を押すと時刻・金額・メモを読み取り専用で表示します。カレンダーを開いても保存データは変更されません。

v0.9.7.1では、カレンダーの肉球を黒背景でも見やすいインラインSVGへ変更しました。日付詳細には、その日の貯金・必要経費・後悔散財・最高の散財を時刻順で表示します。日付セルと月間サマリーは引き続き貯金のみを集計します。

## v0.9.8 貯金目標

1件の「ほしいもの」と目標金額を設定し、目標作成後の貯金で進捗が増えます。目標作成前の金額を含めたい場合は「すでに貯めている金額」へ入力します。目標データは独立キー `chokin-event-app.savingsGoal.v1` に保存され、既存の貯金・出費記録は変更しません。

v0.9.8.1では、目標保存ボタンの視認性、目標達成時の紙吹雪・スキップ、新しい目標への確認導線を改善しました。既存の目標・記録・設定の保存形式は変更していません。

v0.9.8.2では、達成後に続く軽量な星・キラキラ、ホームと詳細で歩くハチワレ、段階的に豪華になる節目アイコンを追加しました。目標入力例は「プレゼント」です。保存形式と進捗計算は変更していません。

v0.9.8.3では、通常画面の猫を固定タブより背面に整理し、達成時の星と紙吹雪を同時開始へ変更しました。猫画像の文字フォールバックを肉球SVGへ置き換え、処理完了後にフェード表示します。保存形式は変更していません。

## v0.9.9 目標達成アルバム

達成済み目標から新しい目標を保存した時、または「アルバムに残して目標を終了」を確定した時だけ、達成当時の目標金額・到達額・達成日などを固定記録として保存します。100％到達や達成演出を閉じただけでは履歴へ追加されません。

達成履歴は現在の目標とは別の `chokin-event-app.goalHistory.v1` に保存されます。JSONバックアップには現在目標と達成履歴の両方を含み、v0.9.8.3以前のバックアップも履歴0件として読み込めます。

## v0.9.9.1 アルバム導線

達成アルバムへの導線はホームには表示せず、履歴が1件以上ある場合だけ目標設定・編集・詳細画面へ表示します。アルバムから戻ると、開く前の目標画面へ戻ります。

## GitHub Pagesで公開する

プロジェクトサイトのサブディレクトリで動く相対パス構成です。推奨リポジトリ名は `chokin-neko-app`、公開URLは `https://＜GitHubユーザー名＞.github.io/chokin-neko-app/` です。ユーザー名をコードへ設定する必要はありません。

公開前に `check_pages_ready.bat` を実行してください。`preview_pages.bat` では `http://localhost:8001/chokin-neko-app/` を開き、GitHub Pagesと同じサブディレクトリ構成を確認できます。詳しい公開手順は `GITHUB_PAGES_DEPLOY.md` を参照してください。

## v0.4 演出素材の差し替え

生成した素材は `assets/cats/`（猫）、`assets/scenes/`（背景・宝箱）、`assets/fx/`（将来の光や粒子素材）へ配置します。形式はPNGまたはWebP、主役画像は透過背景を推奨します。目安は主役が1024×1024px、縦長背景が1080×1920pxです。

ファイル名の例は `cat_celebrate.png`、`cat_surprised.png`、`cat_royal.png`、`treasure_chest_open.png`、`cat_temple_bg.png`、`space_bg.png` です。猫素材は次項の登録ツールを使用してください。対応ファイルがない場合は内蔵SVGへ自動的にフォールバックするため、素材未配置でもアプリは動作します。

## 新しい猫を追加する方法

1. PNG形式の猫画像を準備します。
2. `add_cat.bat` をダブルクリックします。
3. 画面に表示される質問へ順番に回答します。

画像は自動的に `assets/cats/` へコピーされるため、手動で移動する必要はありません。元画像は削除・移動されません。

既存の猫19匹は `assets/cats-transparent/` の透過済み画像を直接表示します。透過済み画像がない追加猫や互換経路では従来のCanvas前処理を使用し、処理できない場合は元画像を表示します。

推奨画像は、正方形・1000×1000px前後・猫全体が画像内に収まった背景透過PNGです。市松模様が画像自体に描かれている場合は、完全な透過画像ではない可能性があります。

- `sync_cats.bat`：`cat-catalog.json`からJavaScriptとmanifestを再同期します。
- `validate_cats.bat`：重複、画像不足、manifest漏れ、未登録画像などを検査します。
- 猫情報の正式な管理元は `assets/cats/cat-catalog.json` です。`cat-characters.js`は直接編集しないでください。

## v0.9.11 バックアップ復元プレビュー

- JSONを選択しても直ちに保存データは変更せず、貯金・出費件数、累計貯金額、現在目標、達成アルバム、実績、猫図鑑、ねこコインを確認してから復元できます。
